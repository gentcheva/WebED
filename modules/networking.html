Circuit switching - Lecture 3

A call/session, no dedicated resources.Finds path in the network, reserves capacity at each link as it passes through, for good call quality. Must have a call set up ie this computer and that computer want to 'talk'. How to divide the capacity? Must divide into pieces, if its not used, it is an idle piece - frequency and time division. May divide frequency based on users, or on time - FDM and TDM. Every slot needs 64kbits so the conversation seems to be continuous.

Packet switching - reliable network.

You don't tell the network that 2 machines wish to communicate, the network does not know what you want to do.

  • No pieces - resources used as needed.
  • Congestion - When the amount of data sent into network may exceed the network capacity
  • Statistical multiplexing - Share link among packet statistically, based on demand

Packet vs circuit switching

Packet switching allows more users to use the network.Circuit switching limits the number of users. For data that comes in bursts, when you're not sure how much data there is, and how many users there are and you donâ€™t know their demand, packet switching is ideal.Congestion occurs in packet switching - not a reliable network.

Network performance

How do loss and delay/latency occur?

Packets queue in router buffers. Packets are ready to send, they just must be pushed onto the link.

4 Sources if packets delay

  • Nodal processing - checking bit errors, determine output link
  • Queuing - time waiting at output link for transmission
  • Transmission delay - time to send bits onto link = L/R , L = packet length, R = bandwidth
  • Propagation delay - delay = d/s, d = length of physical length, s = propagation speed medium
  • s and R are very different quantities

Public vs Private

public means it can be accesses by outsiders, private can only be accessed within the implementation. it isn't a good idea to have publiv attributes. Getters and Setters should be provided where necessary.

Static - Class attribute, without that its an instance attribute

Class Constructors

Used to create an instance of the class. It has no return type, not even void. Many consturctors can be provided, with different number and/or different input types. When constructing the one that fits the parameters given is used

Note.java


                
    public class Note {
        @author name of the author
        @date 18/01/2019
        private static final int MAX_NOTE_SIZE = 40; // upper case because it is a class constant
        private final String comment; // instance attribute

        // private constructor because we have a class method to construct the instances instead,
        // because they dont construct it themselves, they use the API - class methods to do that
        private Note (final String text){
                // class constructors dont have a return type - not even void
            this.comment = text;
        }
        // to call this method
        // Note.makeNote(...) -this tell you its a class method
        public static Note makeNote ( final string text){ // class method because it is static
            // for ease of reading, you should have one return statement in methods
            final Note result; // by saying its final, we convince the compiler that a value has been assigned
            if(text.length > MAX_NOTE_SIZE){
                result = null;
            }else{
                result = new Note(text);
            }
                return result
            }

        /**
        * Get the comment of this Note
        * @param name .....
        * @return the comment of this Note
        */

        public String getComment() { //instance method
            return comment
            }
        public void printComment(){
            System.out.println(this.comment);
            }
        // an object class instance toString , like __str__
        // all classes in java are a part of the default object class 

        @Override // this tells that we are over writing a method, and if we make a small typo it will tell us
        public String toString(){
            return getComment();
            }
        }

    public class Main {

        private static final String PROMPT = "please give input";

        public static void main(final String[] args) {

Encapsulation

Objects should be self governing. They should control their own instance variables. Hiding instance variables prevents direct access because direct access is unsafe / dangerous. Encapsulation Makes learning/using the API easier

Arrays

In java arrays are Objects which contain other things. They can only have things of a single data type. You can have an array of primitive type reference values or an array with an object reference values. When creating an array, Java assigns default values:

  • For int array - numeric 0
  • For boolean - false
  • For char - unicode for 0
  • For object - null


    final int[] numbers = new int[10];
    numbers.length;  // gets the length of the array

    final String[] words = new String[5];

    Dog[] dogs = new Dog[3];
    dogs[1].bark()

Inheritance

Inheritance - is designed for maintainability of code , for reuse of general code, to avoid duplication, for a constant design. They follow is_a relationship. Class names should be nouns, so it makes sense grammatically. put common code into a Superclass.

Implements vs Extends

Java Extends: used for Inheritance, when using a Superclass to inherit(extend) its functionality.
Java implements: used for implementing interfaces (a class that can only contain method signatures and fields, used for polymorphism)

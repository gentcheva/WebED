Inheritance

Using two to three sentences, explain inheritance as a class relationship

Inheritance allows for a class extending(inheriting) it to gain the functionality (all of its methods and attributes). The subclass can now use and add on top of the inherited functionality. Main purpose of inheritance is to share common code written in superclass.

Provide and explain three, and no more than three advantages of inheritance.

Code re-usability - Common code is placed in a common abstract class that can be inherited in many places, rather than re-written in those places.
code maintainability - Since code is written in one place (the common abstract class), it only needs to be maintained in that one place.
Data hiding - base class can decide to keep some data private so that it cannot be altered by the derived class

Inheritance provide a clear model structure which is easy to understand without much complexity Using inheritance, classes become grouped together in a hierarchical tree structure Code are easy to manage and divided into parent and child classes.

Abstract classes and interfaces
Explain the main differences between (regular) classes, abstract classes and an interface

Regular classes - can be instantiated ( made into an object ) can final implementation of a class
abstract classes - provide common functionality (API) (methods have implementations ) and cannot be instantiated
interfaces - fully abstract, give just empty methods (API), cannot be instantiated - when implemented the implementor has to override each method.

What is multiple inheritance and what are interfaces?

Multiple inheritance in Java

Multiple inheritance - the capability of creating a single class that inherits(extends) multiple superclasses. However java doesn't support multiple inheritance, because of abiguity of what method to call (if the method name is the same in more than 1 inherited class)

Interface - An interface is a completely "abstract class" that is used to group related methods(API) that are abstract. When a class implements the interface it MUST override all its methods.

Multiple inheritance is possible in Interfaces, they can inherit many superclasses as the methods are implemented in the class that inherits the interface, so even if interface inherits many X name method, the final class only overrides one X method.

Explain how to use delegation to implement the work() behaviour of a carpenter object. Your design should involve a Worker interface, a ConcreteWorker class, and a Carpenter class. Use Java statements to clarify your example

answer:
Delegation is passing off duty(functionality) to another class. So basically using another class inside a class to receive its functionality to use it to accomplish its own functionality ( without the client knowing we are borrowing functionality) 


public interface worker{
    public void work() ; 
}

public class ConcreteWorker implements worker{
    final String workType;
    public ConcreteWorker(String WorkType){
        workType = workType;
    }
    //...in the class implementation there is this method

    @Override
    public void work(){
        // perform the task
        System.out.println(workType);

    }
}

public class Carpenter {
    // the delegator
    private final ConcreteWorker workBehaviour;

    public Carpenter(String workType){
        workBehaviour = new ConcreteWorker(workType);
    }

    public void work(){
        // using the delegator
        workBehaviour.work();
    }
}

// ... in the main ... 
Carpenter carpenter = new Carpenter();
carpenter.work();   // this uses delegation to implement work functionality but the client isn't aware
